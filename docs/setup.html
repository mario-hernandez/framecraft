FrameCraft MCP

Automate screenshot framing with AI

1Installation

Install the MCP server via Homebrew:

brew install mario-hernandez/framecraft/framecraft-mcp

Verify installation:

framecraft-mcp --version

2Configure Claude Desktop

Open your Claude Desktop configuration file:

~/Library/Application Support/Claude/claude_desktop_config.json

Add the FrameCraft server:

{ "mcpServers": { "framecraft": { "command": "/opt/homebrew/bin/framecraft-mcp", "args": [] } } }

Restart Claude Desktop after saving.

3Available Tools

generate_frame
Generate a single App Store screenshot frame with hero text and gradient background.

Parameters:

ParameterRequiredDescription
screenshot_pathYesAbsolute path to PNG/JPG
hero_textYesMain headline text
subtitleNoSubtitle below hero
templateYesGradient template ID
deviceNoDevice size (default: iphone_6.7)
output_pathYesOutput path (.png)
generate_batch
Generate multiple frames at once.
list_templates
List all available gradient templates.
list_devices
List all available device sizes.

6Usage Examples

Generate a single frame:

"Generate an App Store frame for /Users/me/screenshot.png with the title 'Track Your Progress' using the ocean template. Save it to /Users/me/Desktop/frame.png"

Generate with subtitle:

"Create a frame with hero text 'Beautiful Design' and subtitle 'Made for professionals' using the sunset template for iPhone 6.7"

Generate for MacBook:

"Generate a MacBook Pro frame for my Mac app screenshot using the midnight template"

Batch generation:

"Generate frames for all PNG files in /Users/me/screenshots/ using the berry template. Name them frame_1.png, frame_2.png, etc."

List available options:

"Show me all available FrameCraft templates and devices"

Troubleshooting

Claude doesn't see FrameCraft tools?

1. Make sure you restarted Claude Desktop after editing the config
2. Check that the path is correct: /opt/homebrew/bin/framecraft-mcp
3. On Intel Macs, the path might be: /usr/local/bin/framecraft-mcp

Permission errors?

FrameCraft needs read access to your screenshots and write access to the output folder. Make sure the paths exist and are accessible.
